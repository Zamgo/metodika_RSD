# Popis

Před zahájením ostré tvorby informací musí být projektové metody a postupy prakticky otestovány. Test ověřuje, že dohodnuté postupy pro vytváření, kontrolu, sdílení a předávání informací fungují v reálných podmínkách [[Realizační tým|realizačního týmu]].

Typicky se testuje celý životní cyklus [[Informační kontejner|informačního kontejneru]]: vytvoření dle standardu → interní kontrola kvality → nahrání do [[Společné datové prostředí (CDE)|CDE]] → sdílení ostatním týmům → přezkoumání → schválení. Výsledky testů se dokumentují a případné odchylky od [[BIM Execution Plan (BEP)|BEP]] se řeší.

# Vstupy
- BEP
- Projektové metody a postupy
- Projektový informační standard

# Výstupy
- Protokoly o testování metod a postupů
- Seznam odchylek a nápravných opatření

# Kontrolní body pro správce stavby
- Ověřit, že testy proběhly u všech klíčových procesů
- Zkontrolovat, že odchylky od BEP byly vyřešeny
